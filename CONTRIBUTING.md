Contributing
============

All contributions are welcome, as diagnosing and troubleshooting issues is a challenging task. Please check for open issues before creating new ones, and feel free to open a pull request (PR) for any code changes.

Not all contributions need to be code, the logic and reasoning behind a design is just as valuable. There are a lot of APIs in the Elastic Stack, and sorting through which are the highest value takes time. Figuring how to process and represent the API output as JSON documents to power Kibana visualizations is a great way to contribute.
